# IP Safety Guidelines

Prontera Commerce must become an original commerce world, not a clone of Ragnarok, Pixar, Disney, Ghibli, Animal Crossing, Minecraft, Nintendo, Shopee, Lazada, or any existing franchise or marketplace.

## Legal Safety Principle

Do not copy:

- names
- characters
- maps
- towns
- monsters
- UI
- logos
- music
- sounds
- lore
- distinctive visual identities

## Inspiration vs Copying

Allowed:

- general fantasy market inspiration
- warm town atmosphere
- merchant guild concept
- original cozy commerce world

Not allowed:

- copying recognizable designs
- using franchise names
- using similar mascots
- recreating famous locations
- asking AI tools to generate in the style of named copyrighted studios

## Prompt Safety Rules

Forbidden prompt phrases:

- "Ragnarok style"
- "like Ragnarok Online"
- "Pixar style"
- "Disney style"
- "Ghibli style"
- "Animal Crossing style"
- "Minecraft style"
- "copy this character"
- "make it look like [known IP]"

Preferred prompt phrases:

- "original warm fantasy commerce world"
- "stylized merchant town"
- "cozy artisan marketplace"
- "friendly original shopkeeper character"
- "unique fantasy trade district"
- "soft rounded original 3D character design"
- "family-friendly original fantasy marketplace"

## Asset Review Checklist

Before accepting any art asset, ask:

- Is it original?
- Does it resemble a known IP too closely?
- Are names original?
- Are silhouettes original?
- Are maps original?
- Are UI patterns generic or copied?
- Can we explain the design source independently?

## Human Approval

All public-facing art direction must be reviewed by COM before production use.

AI tools, contractors, teammates, and contributors may propose concepts, but COM / Ponrat / Founder retains final approval authority for public-facing brand and art direction.

## Documentation

All major art decisions should be documented in:

- [Prontera Art Bible](./prontera-art-bible.md)

When a new city, mascot, character family, NPC category, UI motif, or public visual system is approved, document the original source logic so future contributors can continue the style without drifting toward copied IP.
